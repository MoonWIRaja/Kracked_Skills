# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ # SKILL 4: PREMIUM DESIGN SYSTEM # (Merged: UI Constraints + Frontend Design + Anti-Slop) # Scope: Project-wide | File: 04-premium-design-system.md # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ ## When to Apply When making design decisions, building UI components, choosing typography, defining color systems, composing layouts, or reviewing visual quality. ## Design Intentionality ### Before Placing Any Element - Calculate its purpose — if it has no purpose, delete it - Choose a clear aesthetic direction and execute with precision - Reduction is the ultimate sophistication - If it looks like a template or default bootstrap, it is wrong ### What Makes It Memorable - One distinctive choice executed consistently beats five mediocre ones - Bold maximalism and refined minimalism both work — the key is intentionality, not intensity - Every project should feel genuinely designed for its specific context ## Library Discipline (CRITICAL) ### Enforcement Rules - If a UI library (Shadcn UI, Radix, MUI, Headless UI) is detected or active in the project, YOU MUST USE IT - Do not build custom components (modals, dropdowns, buttons, dialogs) from scratch if the library provides them - Do not pollute the codebase with redundant CSS for solved problems - Exception: You may wrap or extend library components to achieve distinctive styling, but the underlying primitive must come from the library for stability and accessibility ### Detection Priority 1. Check package.json for UI libraries 2. Check existing component imports 3. Follow established patterns in the codebase 4. Only build custom when genuinely no library component exists ## Anti-Generic Aesthetics (ENFORCED) ### NEVER Use These Defaults - Font families: Inter, Roboto, Arial, system-ui as display fonts - Color schemes: Purple/violet gradients on white backgrounds - Layouts: Centered hero → 3-card grid → features list → footer - Patterns: Cookie-cutter card grids with identical border-radius - Shadows: Default Tailwind shadow-md on everything ### INSTEAD Pursue - Typography: Distinctive display fonts paired with refined body fonts. Geometric sans, humanist serifs, monospaced accents. Each project should use different type. - Color: Dominant colors with sharp accents outperform timid, evenly-distributed palettes. Commit to a mood. - Layout: Asymmetry, overlap, diagonal flow, grid-breaking elements, generous negative space OR controlled density. - Texture: Gradient meshes, noise/grain overlays, geometric patterns, layered transparencies, decorative borders. ## Visual Hierarchy ### Hierarchy Tools (In Priority Order) 1. **Size** — larger elements draw attention first 2. **Weight** — bold text over regular 3. **Color** — high contrast over muted 4. **Position** — top-left to bottom-right reading flow 5. **Whitespace** — isolation creates importance ### Hierarchy in Practice ```tsx <article> <h1 className="text-3xl font-bold text-gray-900"> Primary Headline </h1> <p className="text-lg text-gray-600 mt-2"> Supporting context with secondary weight </p> <div className="text-base text-gray-500 mt-4"> Body content at tertiary level </div> </article> ``` ## Spacing System (4px Base) ```text 4px - xs (tight, inline elements) 8px - sm (related elements) 16px - md (default spacing) 24px - lg (section spacing) 32px - xl (major sections) 48px - 2xl (page sections) 64px - 3xl (hero spacing) ``` ### Rules - Never use arbitrary values (13px, 17px) - Mobile horizontal padding: 16px minimum - Touch targets: 44x44px minimum ## Typography ### Scale (rem) ```text xs: 0.75rem (12px) - captions, labels sm: 0.875rem (14px) - secondary text base: 1rem (16px) - body text lg: 1.125rem (18px) - emphasized body xl: 1.25rem (20px) - small headings 2xl: 1.5rem (24px) - section headings 3xl: 1.875rem (30px) - page headings 4xl: 2.25rem (36px) - hero headings ``` ### Font Pairing Strategy - Display font: distinctive, characterful (geometric sans, humanist serif, or expressive variable font) - Body font: refined, highly readable - Accent font (optional): monospaced for data/code/labels - NEVER use the same font for all purposes - VARY fonts across projects — do not converge on favorites ### Line Heights and Weights - Headings: 1.2–1.3 line-height, semibold (600) or bold (700) - Body: 1.5–1.6 line-height, regular (400) - Labels/UI: 1.25 line-height, medium (500) ## Color Application ### Semantic System ```css --color-primary: /* main actions, focus states */ --color-text: /* high contrast, readable */ --color-text-muted: /* secondary, supporting */ --color-border: /* subtle separation */ --color-surface: /* card/container backgrounds */ --color-success / --color-warning / --color-error ``` ### Balance Rule - **60%** — dominant (backgrounds, large surfaces) - **30%** — secondary (cards, sections) - **10%** — accent (CTAs, highlights, interactive states) ### Contrast Requirements - Normal text: minimum 4.5:1 ratio - Large text (18px+): minimum 3:1 ratio - Never use color alone to convey meaning ## Layout Composition ### Container Patterns ```tsx // Centered with max-width <div className="mx-auto max-w-4xl px-4 sm:px-6 lg:px-8"> {children} </div> // Full-bleed with contained content <section className="bg-gray-900"> <div className="mx-auto max-w-7xl px-4 py-16"> {children} </div> </section> ``` ### Common Layout Structures ```tsx // Split layout (60/40) <div className="grid grid-cols-1 lg:grid-cols-5 gap-8"> <main className="lg:col-span-3">{content}</main> <aside className="lg:col-span-2">{sidebar}</aside> </div> // Card grid with responsive columns <div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6"> {cards.map(card => <Card key={card.id} {...card} />)} </div> ``` ## Responsive Breakpoints ```text sm: 640px - large phones landscape md: 768px - tablets lg: 1024px - laptops xl: 1280px - desktops 2xl: 1536px - large screens ``` ## Component Hierarchy ### Visual Weight Order 1. Primary action (filled, high contrast) 2. Secondary action (outlined or ghost) 3. Tertiary action (text only, link style) ### Z-Index Scale ```text 0 - base content 10 - dropdowns, popovers 20 - sticky headers 30 - overlays, backdrops 40 - modals, dialogs 50 - toasts, notifications ``` ### Empty States - Explain what belongs here - Guide user to take action - Use friendly, helpful tone - Include clear CTA ### Loading States - Show skeleton shimmer immediately (CSS, not JS) - Load critical content first - Stream in secondary content - Never show stale + loading simultaneously ### Responsive Execution - Design mobile-first, enhance upward - Stack horizontal layouts vertically on mobile - Touch targets: 44x44px with 8px spacing - Thumb-friendly zones for primary actions
