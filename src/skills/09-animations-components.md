# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ # SKILL 9: PREMIUM ANIMATIONS & COMPONENTS # Scope: Dev | File: 09-animations-components.md # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ ## When to Apply When implementing UI animations, selecting or using icon libraries, building reusable components, or adding micro-interactions. ## Animation Philosophy - One well-orchestrated page load with staggered reveals creates more delight than scattered micro-interactions - Use scroll-triggering and hover states that surprise - Match implementation complexity to aesthetic vision: maximalist designs need elaborate animations, minimalist designs need restraint - CSS-only for simple repeating animations, Framer Motion for complex interactive ones ## Icon Libraries ### Primary: Lucide React ```tsx // Always import individual icons import { Search, Menu, X, ChevronRight } from 'lucide-react'; // Standardized wrapper const sizeMap = { sm: 16, md: 20, lg: 24, xl: 32 } as const; interface IconProps { icon: React.ElementType; size?: keyof typeof sizeMap; className?: string; } function Icon({ icon: IconComponent, size = 'md', className }: IconProps) { return ( <IconComponent size={sizeMap[size]} className={className} aria-hidden="true" /> ); } ``` ### Secondary: React Icons (Brand/Social Only) - Use only for icons not available in Lucide - Import individually, never the full set ## Accessibility ```tsx // Decorative: hide from screen readers <Search aria-hidden="true" /> // Interactive: always label the button <button aria-label="Search"><Search aria-hidden="true" /></button> // Meaningful: label the icon <span aria-label="Warning"><AlertTriangle /></span> ``` ## Entry Animations ### Standard Fade-Up ```tsx import { motion } from 'framer-motion'; const fadeUp = { initial: { opacity: 0, y: 8 }, animate: { opacity: 1, y: 0 }, transition: { duration: 0.2, ease: [0.25, 0.46, 0.45, 0.94] }, }; <motion.div {...fadeUp}>Content</motion.div> ``` ### Exit (Always Simpler, Faster) ```tsx <AnimatePresence mode="wait"> {isVisible && ( <motion.div exit={{ opacity: 0 }} transition={{ duration: 0.15 }} > {content} </motion.div> )} </AnimatePresence> ``` ### Staggered Children (Max 8 Items) ```tsx const container = { animate: { transition: { staggerChildren: 0.05, delayChildren: 0.1 }, }, }; const item = { initial: { opacity: 0, y: 8 }, animate: { opacity: 1, y: 0 }, }; <motion.ul variants={container} initial="initial" animate="animate"> {items.slice(0, 8).map((i) => ( <motion.li key={i.id} variants={item}>{i.content}</motion.li> ))} </motion.ul> ``` ## Scroll-Triggered Animations ### Viewport Entry ```tsx <motion.section initial={{ opacity: 0, y: 20 }} whileInView={{ opacity: 1, y: 0 }} viewport={{ once: true, margin: "-100px" }} transition={{ duration: 0.4, ease: "easeOut" }} > Content reveals on scroll </motion.section> ``` ## Micro-Interactions ### Button Press ```tsx <motion.button whileTap={{ scale: 0.97 }} transition={{ duration: 0.1 }} > Click Me </motion.button> ``` ### Hover Card ```tsx <motion.div whileHover={{ y: -2, boxShadow: '0 8px 30px rgba(0,0,0,0.12)', }} transition={{ duration: 0.15 }} > Card Content </motion.div> ``` ### Toggle Switch (Spring Physics) ```tsx <motion.div className="w-12 h-6 rounded-full p-1 cursor-pointer" animate={{ backgroundColor: isOn ? '#22c55e' : '#e5e7eb' }} onClick={() => setIsOn(!isOn)} > <motion.div className="w-4 h-4 bg-white rounded-full shadow" animate={{ x: isOn ? 24 : 0 }} transition={{ type: 'spring', stiffness: 500, damping: 30 }} /> </motion.div> ``` ## Loading States ### Skeleton Shimmer (CSS Only — Saves Bundle) ```css .skeleton { background: linear-gradient( 90deg, #e5e7eb 25%, #f3f4f6 50%, #e5e7eb 75% ); background-size: 200% 100%; animation: shimmer 1.5s infinite; } @keyframes shimmer { 0% { background-position: 200% 0; } 100% { background-position: -200% 0; } } ``` ### Spinner (Actions Only, Never Content Areas) ```tsx <motion.div className="w-5 h-5 border-2 border-white border-t-transparent rounded-full" animate={{ rotate: 360 }} transition={{ duration: 1, repeat: Infinity, ease: 'linear' }} /> ``` ## Component Composition ### Class Variance Authority (cva) ```tsx import { cva, type VariantProps } from 'class-variance-authority'; const buttonVariants = cva( 'inline-flex items-center justify-center rounded-md font-medium transition-colors focus-visible:outline-none focus-visible:ring-2', { variants: { variant: { primary: 'bg-primary-600 text-white hover:bg-primary-700', secondary: 'bg-gray-100 text-gray-900 hover:bg-gray-200', ghost: 'hover:bg-gray-100 text-gray-700', destructive: 'bg-red-600 text-white hover:bg-red-700', }, size: { sm: 'h-8 px-3 text-sm', md: 'h-10 px-4 text-sm', lg: 'h-12 px-6 text-base', }, }, defaultVariants: { variant: 'primary', size: 'md' }, } ); interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement>, VariantProps<typeof buttonVariants> {} const Button = forwardRef<HTMLButtonElement, ButtonProps>( ({ className, variant, size, ...props }, ref) => ( <button ref={ref} className={cn(buttonVariants({ variant, size }), className)} {...props} /> ) ); ``` ## Performance Rules - NEVER animate width, height, top, left — use transforms - Use will-change sparingly, remove after animation - CSS for simple repeating animations, Framer for interactive - Import only specific icons, never the full library
